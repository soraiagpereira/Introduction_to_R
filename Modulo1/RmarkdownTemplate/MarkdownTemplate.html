R Markdown

Este um documento de R Markdown, que tem a exteno .Rmd. O Markdown uma sintaxe de formatao para criar facilmente documentos HTML. Na realidade, permite tambm criar PDFs e at documentos de Word, mas no nos vamos abordar esses formatos neste template. Estes HTML sero documentos dinmicos, na medida em que se baseiam em dados, e quando os dados mudam, os documentos mudam tambm. Sero apresentados exemplos deste comportamento dinmico baixo.

Para mais detalhes sobre o uso do R Markdown existem muito recursos online. Destes salientamos:

  • http://rmarkdown.rstudio.com Analyze. Share. Reproduce. Your data tells a story. Tell it with R Markdown. Turn your analyses into high quality documents, reports, presentations and dashboards;
  • https://bookdown.org/yihui/bookdown/ Como escrever livros inteiros em Markdown, bookdown: Authoring Books and Technical Documents with R Markdown por Yihui Xie;
  • https://rstudio.com/wp-content/uploads/2015/02/rmarkdown-cheatsheet.pdf Cheat Sheet sobre R Markdown.

As possibilidades so imensas, mas podemos facilmente formatar o texto, e.g.:

  1. itlico: itlico
  2. bold: bold
  3. bold e itlico: bold

Quando pressionamos o boto Knit dentro do RStudio, o .Rmd vai ser compilado, gerando um .html (html por defeito) com o mesmo nome.

Este documento que est a ler disponibilizado como um template. Ao comparar este .Rmd com o .html que criado a partir dele poder perceber como funciona o R Markdown.

Note-se em particular que o resultado de um relatrio dinmico vai poder incluir:

  • contedo escrito
  • cdigo R
  • output de cdigo R
  • conteudo externo, e.g.figuras externas

Isto que est a ler conteudo escrito e o cdigo encontra-se no que se denominam code chuncks. O output do cdigo s o v depois de compilar o documento (experimente faz-lo depois de ler isto). Para criar um chunk de cdigo temos de o indicar assim:

summary(beaver1)
##       day             time             temp           activ

##  1st Qu.:346.0   1st Qu.: 932.5   1st Qu.:36.76   1st Qu.:0.00000  
##  Median :346.0   Median :1415.0   Median :36.87   Median :0.00000  
##  Mean   :346.2   Mean   :1312.0   Mean   :36.86   Mean   :0.05263  
##  3rd Qu.:346.0   3rd Qu.:1887.5   3rd Qu.:36.96   3rd Qu.:0.00000  
##  Max.   :347.0   Max.   :2350.0   Max.   :37.53   Max.   :1.00000

Note-se que o output foi criado pela funo do R summary, que permite visualizar um sumrio do objecto que colocamos como seu argumento (ou seja, o argumento o que est dentro dos parentesis!). O chunk acima mostra um sumrio do objecto badger1 sobre um conjunto de dados que vem por default na instalao base do R. Se quiser saber mais sobre este conjunto de dados, do package datsets, basta chamar a ajuda do R. Para tal poder escrever na linha de comandos ?badger1. Esta forma de pedir ajuda vlida para qualquer tipo de objecto ou funo do R. Experimente!

Se quisermos introduzir uma figura externa, podemos faz-lo de diversas maneiras, mas esta a mais simples

esta figura foi retirada de um post sobre R: https://towardsdatascience.com/a-complete-guide-to-learn-r-29e691c61d1

Para incluir um grfico de R no seu relatrio dinmico basta criar um grfico num chunck, por exemplo, assim

plot(beaver1$time,beaver1$temp)

neste grfico temos representada a temperatura de um castor ao longo de cerca de dois dias. Repare que se quisermos, podemos ter no relatrio dinmico valores dinmicos, ou seja, que se alteram quando os dados so alterados. Por exemplo, o dobro da temperatura mnima do castor foi de 72.66 e a temperatura mxima foi 37.53, sendo que a sua mdia foi 36.862193. Note-se que podemos arredondar os valores para ficarem mais bonitos no texto, por exemplo, a mdia aparecer como 36.9.

Veja no .Rmd como para criar estes valores dinmicos temos de usar r expresso dentro de aspas (num teclado em portugs, estas aspas correspondem ao acento grave) onde a expresso o cdigo de R que gera o valor.

De notar que a opo echo = FALSE pode ser adicionada a um chunk para que o cdigo no seja visualizado, mas o seu output sim

Se est a ler o .html, apesar de ver um grfico acima (um histograma de 100 nmeros pseudo-aleatrios Gaussianos), no vmos o cdigo que o gerou, esse s se consegue ver no .Rmd.

Vejamos agora um exemplo do dinamismo destes relatrios. Imagine que geramos 10 numeros pseudo-aleatrios Gaussianos

set.seed(123)#use ?set.seed para descobrir o que faz esta funo!
x <- rnorm(100)

Podemos fazer a soma destes nmeros

sum(x)
## [1] 9.040591

Neste momento, se alterarmos os dados que esto no objecto x, por exemplo substituindo o primeiro valor por um novo nmero

x[1] <- 100

o mesmo cdigo ir criar um novo output

sum(x)
## [1] 109.6011

uma vez que os dados mudaram.

Agora podemos antecipar uma das grandes vantagens dos documentos dinmicos. cada vez que os seus dados base mudarem, j no tem de repetir o seu relatrio, basta recompilar o documento, e j est - tudo actualizado para a ultima verso dos dados!

Se quisermos ver cdigo mas sem este ser executado, til por exemplo se tivermos um erro no cdigo, podemos usar a opo eval = FALSE, como por exemplo

#este cdigo tem um erro: devia ser plot e no prot!
prot(rnorm(100))

Note-se que o # tem um efeito diferente no texto corrido e nos chunks:

  • em chuncks, processado pelo R, logo interpretado como um comentrio, que no portanto implementado ( mesmo um comentrio, apenas os humanos o conseguem ler)
  • no texto, indica um cabealho, de nvel 1 #, nvel 2 ##, nvel 3 ###, e assim sucessivamente.

Assim sendo, vamos de seguida criar seces e sub-seces para ilustrar esta capacidade.
